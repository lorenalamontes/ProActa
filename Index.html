Paso 1 — Preparación de datos

Pipeline ETL

El archivo Excel de Mundipartes (6.656 filas, 16 columnas) fue procesado mediante un pipeline ETL que transforma los datos dispersos en una tabla analítica lista para el modelo.

Extract

Extracción

  • Archivo Demanda Proyecto IA.xlsx
  • 6.656 registros · 16 columnas
  • Columnas: Asesor, Cliente ajustado, Marca, Artículo + 12 meses (jun 2025 – may 2026)
  • Conversión de columna Artículo a tipo String
Transform

Transformación

  • Todos los NaN → 0 (fillna)
  • Unpivot: formato ancho → largo (pd.melt)
  • Filtro: solo transacciones reales (Cantidad > 0)
  • Feature Engineering: Demanda_Promedio, Demanda_Total, Frecuencia_Compras, Ciclo_Consumo_Dias, Stock_Seguridad, Pedido_Sugerido_Proactivo, Valor_Oportunidad
  • Segmentación ABC por porcentaje acumulado de demanda
Load

Carga

  • 6.410 pares únicos Cliente-Artículo
  • DataFrame df_agg listo para ML
  • Central de Venta Consultiva interactiva por cliente
  • Tabla de resultados para el asesor comercial
ETL completo · celda 4
# Unpivot: transformar columnas de meses en filas columnas_meses = [col for col in df.columns if '-' in str(col)] df_melted = pd.melt(df, id_vars=cols_id, value_vars=columnas_meses, var_name='Periodo', value_name='Cantidad') # Limpiar y filtrar solo transacciones reales df_melted['Cantidad'] = pd.to_numeric(df_melted['Cantidad'], errors='coerce').fillna(0) df_melted = df_melted[df_melted['Cantidad'] > 0] # Agregación por Cliente-Artículo df_agg = df_melted.groupby(['Asesor', 'Cliente ajustado', 'Marca', 'Articulo']).agg( Demanda_Promedio = ('Cantidad', 'mean'), Demanda_Total = ('Cantidad', 'sum'), Frecuencia_Compras= ('Periodo', 'count') ).reset_index() # Feature engineering comercial df_agg['Ciclo_Consumo_Dias'] = (365 / df_agg['Frecuencia_Compras']).round(0) df_agg['Stock_Seguridad'] = (df_agg['Demanda_Promedio'] * 0.20).round(0) df_agg['Pedido_Sugerido_Proactivo'] = (df_agg['Demanda_Promedio'] + df_agg['Stock_Seguridad']).round(0) # ✅ ¡Datos procesados correctamente! # 📊 Total de registros únicos Cliente-Artículo: 6410

Paso 3 — Machine Learning

Selección y Entrenamiento del Modelo

Se evaluaron tres algoritmos de regresión. El Gradient Boosting Regressor fue seleccionado por su capacidad de capturar relaciones no lineales en los patrones de consumo.

Regresión Lineal Baseline

Modelo simple e interpretable. Útil como referencia de comparación.

✓ Ventajas: Rápida, interpretable, sin hiperparámetros complejos.
✗ Limitación: No captura relaciones no lineales entre artículo y frecuencia.
Árbol de Decisión Evaluado

Captura reglas de decisión claras. No le afectan los valores atípicos.

✓ Ventajas: Relaciones no lineales, reglas visualizables.
✗ Limitación: Propenso a sobreajuste si el árbol crece demasiado.
Gradient Boosting ✓ Seleccionado

Ensamble de árboles que corrige errores iterativamente. Reduce sobreajuste frente al árbol simple.

✓ Ventajas: Robusto, captura patrones complejos, resistente a outliers.

Paso 4 — Inteligencia Comercial

Segmentación ABC de Clientes

La clasificación ABC prioriza los clientes según su participación en el volumen total de demanda. El 80% de la demanda es generado por el segmento A.

A
Estratégico
≤ 80% acumulado · Alta prioridad

Clientes que concentran el 80% del volumen total de demanda. Alta frecuencia de compra, ciclos predecibles y mayor valor de oportunidad por pedido.

Contacto proactivo semanal
B
En Desarrollo
81–95% acumulado · Potencial medio

Clientes con compras moderadas. Tienen potencial de crecimiento con acompañamiento activo del asesor y propuestas de pedido sugerido.

Plan de activación mensual
C
Bajo Impacto
> 95% acumulado · Revisar estrategia

Clientes con baja demanda o inactividad. Evaluar costo de atención frente al valor generado. Candidatos a estrategia digital o re-activación.

Re-evaluar / automatizar
Clasificación ABC · celda 4
# Calcular porcentaje acumulado de demanda df_agg = df_agg.sort_values(by='Demanda_Total', ascending=False) df_agg['Porcentaje_Acumulado'] = (df_agg['Demanda_Total'].cumsum() / df_agg['Demanda_Total'].sum()) * 100 def clasificar_abc(pct): if pct <= 80: return 'A (Estratégico)' elif pct <= 95: return 'B (Desarrollo)' else: return 'C (Bajo Impacto)' df_agg['Segmento_Cliente'] = df_agg['Porcentaje_Acumulado'].apply(clasificar_abc)

Resultados y Aprendizajes

Conclusiones

Qué aprendimos del modelo, qué limitaciones encontramos y cuál es el camino de mejora para convertir ProActa en un sistema robusto de venta consultiva.

OT-C01 · ETL y Datos

El proceso ETL transformó exitosamente 6.656 registros dispersos en 6.410 pares únicos Cliente-Artículo con variables derivadas de inteligencia comercial. La transformación de formato ancho a largo (melt) fue clave para extraer la frecuencia real de compra por periodo.

OT-C02 · Segmentación ABC

La clasificación ABC identificó con precisión los clientes estratégicos (segmento A) que concentran el 80% del volumen de demanda. Esta priorización permite al asesor enfocar su tiempo en los contactos de mayor retorno antes de que el cliente lo solicite.

OT-C03 · Gradient Boosting

El modelo Gradient Boosting con 120 estimadores obtuvo MAE de 34.81 unidades y un R² de −0.0219. El R² negativo señala que el modelo en su estado actual no supera un predictor base simple, indicando que los features actuales requieren enriquecimiento con historial de meses reales por cliente.

OT-C04 · Central Consultiva

La Central de Venta Consultiva interactiva es el entregable más valioso para el equipo comercial: permite seleccionar un cliente y visualizar su perfil de consumo, ciclo de compra, pedido sugerido y valor de oportunidad, facilitando el contacto proactivo con datos concretos.
